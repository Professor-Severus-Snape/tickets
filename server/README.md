# Backend

Серверная часть приложения «Менеджер тикетов».  
Описание проекта — в [главном README](../README.md).

## Назначение

- Обработка HTTP-запросов
- Хранение и обновление тикетов
- Создание, редактирование и удаление задач
- Возврат данных в формате JSON

## API

Сервер предоставляет HTTP API для работы с тикетами.

### Структура данных

Каждый тикет имеет следующую структуру:

```javascript
Ticket {
  id          // уникальный идентификатор
  name        // краткое описание задачи
  status      // boolean: выполнена или нет
  description // полное описание задачи
  created     // дата создания (timestamp)
}
```

### Основные эндпоинты

- `GET ?method=allTickets` - получить список всех тикетов (без описания)
- `GET ?method=ticketById&id=<id>` - получить полный объект тикета по id
- `POST ?method=createTicket` - создать новый тикет
- `POST ?method=updateById&id=<id>` - обновить существующий тикет по id
- `GET ?method=deleteById&id=<id>` - удалить тикет по id

## Технологии

- Node.js
- Express
